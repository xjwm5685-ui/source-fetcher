## 🚀 How It Works

### Choco Dependency Handling

```bash
# Install choco package
# Chocolatey automatically handles dependencies
sfer install --source choco --name git
```

**Implementation**:
- Download `.nupkg` file
- Execute `choco install package.nupkg --yes`
- Chocolatey resolves and installs dependencies automatically
- Track installation in unified manifest

### Winget Dependency Handling

```bash
# Install winget package
# Windows installer handles dependencies
sfer install --source winget --name Microsoft.PowerToys
```

**Implementation**:
- Download installer (.exe/.msi/.msix)
- Execute with silent parameters
- Installer handles its own dependencies
- Track installation in unified manifest

### URL Dependencies

```yaml
downloads:
  - source: url
    url: https://example.com/app.exe
    dependencies:
      - url: https://example.com/runtime.exe
        name: "Required Runtime"
        version: "2.0"
      - url: https://example.com/lib.dll
        name: "Core Library"
```

**Implementation**:
- Download main file
- Download all declared dependencies
- Track all files in manifest
- Generate dependency report

### Unified Uninstall

```bash
# Uninstall specific package
sfer uninstall --source choco --name git

# Uninstall all from one ecosystem
sfer uninstall --all --source choco

# Uninstall everything
sfer uninstall --all

# Interactive mode
sfer uninstall --all --interactive

# Dry run
sfer uninstall --all --dry-run
```

**Implementation**:
- Read unified manifest
- Filter by source/package
- Execute appropriate uninstall method
- Update manifest
- Report results

## 💡 Design Decisions

### Why Let Choco/Winget Handle Dependencies?

**Pros**:
- Simpler implementation
- Matches official behavior
- More reliable
- Less maintenance

**Cons**:
- Less control
- Requires native tools installed
- Can't preview dependency tree

**Decision**: Use native tools for v1.1.0, consider custom resolver for v1.2.0 if needed.

### Why Manual URL Dependency Declaration?

**Pros**:
- User has full control
- Simple to implement
- Flexible for any scenario

**Cons**:
- User must manually specify dependencies
- No automatic discovery

**Decision**: Manual declaration for v1.1.0, explore auto-detection in future.

### Why Unified Manifest?

**Pros**:
- Single source of truth
- Cross-ecosystem tracking
- Simpler uninstall logic

**Cons**:
- More complex format
- Need migration from v1.0.0

**Decision**: Use unified format, provide migration tools for v1.0.0 manifests.
